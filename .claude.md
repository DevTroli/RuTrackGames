# .claude.md

This file provides additional guidance for Claude Code when working with the rtlinux project.

## Code Style

### Structure
- Maintain single-file organization (`rtlinux.py`)
- Use section headers: `# ── Section Name ─────────────────────────────────────`
- Place constants near top, functions in logical order (imports → constants → layers → main)

### Naming
- snake_case for functions and variables
- CONSTANT_CASE for configuration values (e.g., `CONCURRENCY`, `PAGE_SIZE`)
- PascalCase only for class names (if any)

### Comments
- Single-line comments: `# Comment text` (space after #)
- Section separators: `# ── Name ──` style
- Docstrings: Minimal, focus on "what" not "how"

### Rich Styling
- Styles in tags: `[bold cyan]text[/bold cyan]`
- Use `Text` object for markup escaping
- Preferred styles: `[dim]`, `[yellow]`, `[red]`, `[green]`, `[bold]`, `[blue]`, `[cyan]`

## Security Checklist

Before any change that touches:
- 🔒 SSL/TLS settings → ensure `certifi.where()` context is used
- 🔒 URL handling → validate scheme and domain before `webbrowser.open()`
- 🔒 User input → escape Rich markup with `Text()` or position args

## Cache Handling

- Always use `with open(...)` for file I/O
- Validate JSON before using (list of dicts with required keys)
- Corrupt cache → auto-re-fetch, never `sys.exit()`
- Cache location: `~/.cache/rtlinux/` (XDG) or `%LOCALAPPDATA%/rtlinux/` (Windows)

## Async Patterns

- Use `@async` functions with `asyncio.gather()` for parallel fetches
- Semaphore for rate limiting: `asyncio.Semaphore(CONCURRENCY)`
- Use `asyncio.get_running_loop()` (NOT deprecated `get_event_loop()`)
- Catch specific exceptions: `(aiohttp.ClientError, asyncio.TimeoutError, OSError)`

## Testing (Future)

When tests are added:
- Unit tests for pure functions: `search_with_strategies()`, `format_size()`, `parse_size()`
- Integration tests for HTTP layer (mocked)
- No need to test Rich display rendering

## Common Patterns

### Adding a new forum
1. Add to `FORUM_SUBFORUMS` dict
2. Add to appropriate platform list (`WINDOWS_FORUMS`, etc.)
3. Update `KNOWN_FORUMS` if needed

### Adding a new search strategy
1. Create function with signature: `(catalog, query) -> list`
2. Add to `search_with_strategies()` loop
3. Return strategy name for display

### Adding a new TUI command
1. Create `cmd_name(state, catalog)` function
2. Add to `COMMANDS` dict (no args) or `PREFIX_COMMANDS` (with args)
3. Update welcome panel with help text
